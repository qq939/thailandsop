# 指纹模块 Modbus RTU 说明文档

本文档根据供应商截图整理，用于后续开发串口 Modbus 指纹模块类。

## 1. 接线定义

模块侧接线如下：

| 线色 | 描述 | 符号 |
| --- | --- | --- |
| 红 | 电源正极 | VCC |
| 黑 | 电源负极 | GND |
| 黄 | RS485 A | A |
| 绿 | RS485 B | B |

注意事项：

- 必须严格按引线规定接线，否则可能损坏设备。
- 工作时严禁线路之间短路。
- 模块外置型按颜色对应连接：
  - 控制器：棕 / 白 / 蓝 / 灰
  - 模块：红 / 黑 / 黄 / 绿

## 2. 通信协议

| 项目 | 值 |
| --- | --- |
| 通信接口 | RS485 / RS232 |
| 通信协议 | Modbus RTU |
| 通信格式 | 9600, 8, N, 1 |
| 默认 Modbus 地址 | 1，或 255 广播地址 |

开发默认值：

- 波特率：`9600`
- 数据位：`8`
- 校验位：`None`
- 停止位：`1`
- 默认从站地址：`1`
- 广播地址：`255`

## 3. 功能概览

模块主要支持：

- 指纹识别模式
- 指纹录入模式
- 指纹验证
- 删除指纹
- 清空指纹库
- 查询有效指纹数量
- 查询模块状态
- 光环控制

默认情况下，模块处于指纹识别模式，等待手指放到采集区域。成功获取一次指纹图像后，蜂鸣器会短鸣一声。

## 4. 寄存器定义

### 4.1 数据寄存器

| 地址 | 访问 | 功能 | 参数 / 返回值 | 功能码 | 备注 |
| --- | --- | --- | --- | --- | --- |
| `0000H` | 写 | 启动指纹录入 | 指纹存储 ID | `06` | ID 范围：`1` 到 `255` |
| `0001H` | 写 | 取消操作 | `01` 表示取消 | `06` | 取消录入流程，退出循环并进入搜索模式；也可退出控制流程并进入搜索模式 |
| `0002H` | 写 | 删除指纹 | 要删除的指纹 ID | `06` | 成功返回正常写响应，失败返回失败原因 |
| `0003H` | 写 | 清空指纹库 | `01` 表示清空 | `06` | 删除所有已录入指纹 |
| `0004H` | 写 | 光环控制 | 高字节为灯光模式，低字节为颜色 | `06` | 默认值为 `0`，例如 `0x0101` 表示红色呼吸灯 |
| `0010H` | 读 | 指纹识别结果 / 搜索结果 | 成功返回搜索到的 ID，失败返回 `83 + 结果` | `03` | 结果寄存器 |
| `0011H` | 读 | 搜索指纹分数 | 分数 | `03` | 作为结果参考 |
| `0012H` | 读 | 有效模板数量 | 数量 | `03` | 已录入的有效指纹模板个数 |
| `0013H` | 读 | 模块状态 | 状态码 | `03` | 见“模块状态”章节 |

### 4.2 通信寄存器

| 地址 | 访问 | 功能 | 默认值 | 功能码 | 备注 |
| --- | --- | --- | --- | --- | --- |
| `1000H` | 读 / 写 | Modbus 从站地址 | `1` | `03`, `06` | 读取或修改模块 Modbus 地址 |

## 5. 模块状态

读取 `0013H` 寄存器可获取模块当前状态：

| 值 | 含义 |
| --- | --- |
| `01` | 指纹录入 |
| `02` | 指纹识别 |
| `03` | 交互操作中 |

供应商说明中提到，交互操作包括删除指纹、清空指纹、光环控制、读取有效指纹数量等操作。

## 6. 光环控制

通过 `06` 功能码写入 `0004H` 寄存器。

参数格式：

- 高字节：灯光模式
- 低字节：灯光颜色

灯光模式：

| 高字节 | 模式 |
| --- | --- |
| `01` | 呼吸灯 |
| `02` | 闪烁灯 |
| `03` | 灯常开 |
| `04` | 灯常灭 |
| `05` | 灯渐开 |
| `06` | 灯渐灭 |

灯光颜色：

| 低字节 | 颜色 |
| --- | --- |
| `01` | 红色 |
| `02` | 蓝色 |
| `03` | 紫红色 |

示例：

- `0x0101`：红色呼吸灯
- `0x0302`：蓝色常亮
- `0x0401`：关闭灯光。颜色字节可能会被模块忽略，但在硬件验证前建议仍传入合法颜色值。

## 7. 操作流程

### 7.1 指纹识别

指纹识别是模块默认模式。

1. 确认模块不在指纹录入流程中。
2. 用户将手指按在采集区域。
3. 模块采集图像并进行识别。
4. 上位机通过 `03` 功能码读取 `0010H`。
5. 成功时，`0010H` 返回匹配到的指纹 ID。
6. 失败时，供应商描述为返回 `83 + 结果`。
7. 可选读取 `0011H` 获取匹配分数。

时间注意事项：

- 识别结果 ID 会保持约 5 秒，之后清除为 `FFFFH`。
- 上位机应在手指触发后及时读取识别结果。

### 7.2 指纹录入

通过 `06` 功能码向 `0000H` 写入目标指纹 ID，启动录入流程。

1. 上位机写入 `0000H = 指纹ID`。
2. 模块进入指纹录入流程。
3. 用户将手指按在指纹采集区域。
4. 录入成功后，蜂鸣器长鸣一声。
5. 录入失败后，模块等待重新录入。
6. 需要取消时，写入 `0001H = 01`；部分特殊型号也可按 `K4` 取消。

特殊型号补充：

- 按键 `K1`、`K2`、`K3` 可分别启动存储 ID 为 `1`、`2`、`3` 的指纹录入流程。

### 7.3 指纹验证

供应商资料中将“指纹验证”和默认识别/搜索做了区分。

1. 确认模块不处于指纹录入流程。
2. 用户将手指按在采集区域。
3. 验证成功后，蜂鸣器长鸣一声。
4. 供应商描述中提到通过 `03` 功能码读取 `0011H` 获取结果。

待验证点：

- 截图文字说读取 `0011H` 获取结果 ID，但寄存器表中 `0011H` 是比对结果分数，`0010H` 才是识别结果 ID。开发时先按 `0010H` 为主结果 ID、`0011H` 为分数处理，等实测硬件后再修正。

### 7.4 删除指纹

1. 上位机通过 `06` 功能码写入 `0002H = 指纹ID`。
2. 成功时返回正常 Modbus 写响应。
3. 失败时返回失败原因。

### 7.5 清空指纹库

1. 上位机通过 `06` 功能码写入 `0003H = 01`。
2. 模块清空所有已录入指纹。

### 7.6 查询有效指纹数量

1. 上位机通过 `03` 功能码读取 `0012H`。
2. 返回值为当前已录入的有效指纹模板数量。

### 7.7 查询模块状态

1. 上位机通过 `03` 功能码读取 `0013H`。
2. 根据“模块状态”章节解释返回值。

### 7.8 修改 Modbus 地址

1. 上位机通过 `06` 功能码写入 `1000H = 新地址`。
2. 截图中未明确普通地址范围。开发时建议按标准 Modbus RTU 使用 `1` 到 `247`，除非实测确认模块支持 `255` 作为普通通信地址。
3. 写入后新地址是否立即生效，或需要重新上电，需要硬件验证。

## 8. 建议的软件接口

后续可封装一个指纹模块服务类，例如：

```csharp
public interface IFingerprintModule
{
    Task StartEnrollmentAsync(byte fingerprintId, CancellationToken ct);
    Task CancelAsync(CancellationToken ct);
    Task DeleteFingerprintAsync(byte fingerprintId, CancellationToken ct);
    Task ClearDatabaseAsync(CancellationToken ct);
    Task<FingerprintRecognitionResult> ReadRecognitionResultAsync(CancellationToken ct);
    Task<int> ReadTemplateCountAsync(CancellationToken ct);
    Task<FingerprintModuleStatus> ReadStatusAsync(CancellationToken ct);
    Task SetLightAsync(FingerprintLightMode mode, FingerprintLightColor color, CancellationToken ct);
    Task SetSlaveAddressAsync(byte address, CancellationToken ct);
}
```

建议的领域类型：

```csharp
public enum FingerprintModuleStatus
{
    Enrollment = 1,
    Recognition = 2,
    InteractiveOperation = 3
}

public enum FingerprintLightMode
{
    Breathing = 0x01,
    Flashing = 0x02,
    AlwaysOn = 0x03,
    AlwaysOff = 0x04,
    FadeOn = 0x05,
    FadeOff = 0x06
}

public enum FingerprintLightColor
{
    Red = 0x01,
    Blue = 0x02,
    Purple = 0x03
}

public sealed record FingerprintRecognitionResult(
    bool Success,
    int? FingerprintId,
    int RawValue,
    int? Score,
    string? FailureReason);
```

## 9. 开发注意事项

- 读取寄存器使用 Modbus RTU `03` 功能码。
- 写单个寄存器使用 Modbus RTU `06` 功能码。
- Modbus 寄存器值按大端序传输。
- 轮询间隔建议控制在 `100` 到 `300 ms`，避免过度占用 RS485 总线。
- 识别结果约 5 秒后会清除，识别流程中应主动轮询，不能太慢。
- 在实测确认前，将 `FFFFH` 按“当前无结果 / 结果已清除”处理。
- 串口异常、CRC 异常、超时异常不要静默吞掉，应向上层返回明确错误并记录诊断日志。
- 写入指纹 ID 前应校验范围。供应商给出的录入 ID 范围为 `1` 到 `255`。
- 不建议对广播地址执行读操作。按 Modbus 语义，广播通常只适合写操作。

## 10. 硬件验证项

以下内容图片资料中不够明确，需要拿到实物后验证：

- `0010H` 失败返回值中 `83 + 结果` 的具体含义。
- `0011H` 是否始终表示分数，还是在验证模式下也可能表示结果 ID。
- `FFFFH` 是由 `0010H` 返回，还是 `0011H` 返回，或两者都会返回。
- 地址 `255` 是否只能作为广播地址，还是也可作为普通从站地址。
- 写入 `1000H` 修改地址后，是立即生效还是重新上电后生效。
- 当前选型模块实际使用 RS485、RS232，还是两种接口均支持。
